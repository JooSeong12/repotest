1. Internet
   - 인터넷은 어떻게 작동하나? : 일련의 표준화된 프로토콜을 사용하여 장치와 컴퓨터 시스템을 함께 연결함으로써 작동, 프로토콜은 장치 간에 정보가 교환되는 방식을 정의하고
                                             데이터가 안정적이고 안전하게 전송되도록 보장
                                             인터넷의 핵심은 서로 다른 장치와 시스템 간의 트래픽 전달은 담당하는 상호 연결된 라우터의 글로벌 네트워크
                                             라우터는 패킷을 검사하여 목적지를 향한 경로의 다음 라우터로 전달 이 프로세스는 패킷이 최종 목적지에 도달할 때까지 계속 된다.
                                             패킷이 올바르게 전송되고 수신되도록 TCP/IP 프로토콜을 사용
                                             TCP(전송제어프로토콜) : 패킷이 올바른 순서로 안정적으로 전송되도록 보장
                                             IP(인터넷 프로토콜) : 패킷이 올바른 대상으로 라우팅하는 역할을 담당
   - 인터넷의 기본 개념 및 용어
      패킷 : 인터넷을 통해 전송되는 작은 데이터 단위
      라우터 : 서로 다른 네트워크 간에 데이터 패킷을 전달하는 장치
      IP주소 : 네트워크의 각 장치에 할당된 고유 식별자로 데이터를 올바른 대상으로 라우팅하는데 사용
      도메인 이름 : google.com과 같이 웹사이트를 식별하는 데 사용되는 사람이 읽을 수 있는 이름
      DNS : 도메인 이름 시스템의 도메인 이름을 IP 주소로 변환하는 역할을 담당
      HTTP(HyperText Transfer Protocol) : 클라이언트(웹 브라우저)와 서버(웹 사이트)간에 데이터를 전송하는데 사용
                                                       클라이언트 : 클라이언트는 사용자가 직접 조작하며, 서버에 요청을 보내고 서버로부터 응답을 받는 컴퓨터나 소프트웨어 ex)웹 브라우저, 모바일 앱
                                                       서버 : 클라이언트의 요청을 처리하고, 필요한 데이터를 제공하는 컴퓨터나 소프트웨어 ex) 웹 서버, 데이터베이스 서버
      HTTPS : 클라이언트와 서버 간의 보안 통신을 제공하는데 사용되는 암호화된 HTTP 버전
      SSL(Secure Sockets Layer)/TLS(Transport Layer Security) : 인터넷을 통한 보안 통신을 제공하는데 사용  

   - 프로토콜의 역할
     프로토콜의 정의 : 장치와 시스템 간에 정보가 교환되는 방식을 정의하는 일련의 규칙 및 표준
     IP(Internet Protocol) : 데이터 패킷을 올바른 대상으로 라우팅 하는 역할
     TCP(Transmission Control Protocol) : 데이터 패킷을 세그먼트로 나누어 전송하고, 수신 측에서 이를 재조립하며, 손실된 패킷을 재전송     
     UDP(User Datagram Protocol) : TCP와 반대로 비연결형이며 빠른 전송속도가 특징이고 데이터 손실있을 수 있다.
     HTTP (HyperText Transfer Protocol) : 웹 브라우저와 웹 서버 간의 통신을 담당, 클라이언트의 요청과 서버의 응답을 통해 웹 페이지를 전송
     FTP (File Transfer Protocol) : 네트워크를 통해 파일을 전송하는 프로토콜, 파일 업로드와 다운로드, 디렉토리 관리 등을 지원
     SMTP (Simple Mail Transfer Protocol) : 이메일 전송을 담당하는 프로토콜
     DNS (Domain Name System) : 도메인 이름을 IP 주소로 변환하는 시스템
     DHCP (Dynamic Host Configuration Protocol) : 네트워크 장치에 동적으로 IP 주소를 할당하는 프로토콜 / IP 주소, 서브넷 마스크, 게이트웨이 등을 자동으로 할당
     SNMP (Simple Network Management Protocol) : 네트워크 장치를 모니터링하고 관리하기 위한 프로토콜

   - IP 주소 및 도메인 이름 이해
     IP 주소 : 네트워크의 각 장치에 할당된 고유 식별자, 데이터를 올바른 대상으로 라우팅하여 정보가 의도한 수신자에게 전송되도록 하는데 사용
     도메인 : 웹사이트와 기타 인터넷 리소스를 식별하는데 사용되는 사람이 읽을 수 있는 이름 도메인 이름은 DNS를 통해 IP주소로 변환

   - HTTP 및 HTTPS 소개
     HTTP : 클라이언트와 서버간에 데이터를 전송하는데 사용되는 프로토콜 / 웹사이트를 방문하면 웹 브라우저는 서버에 HTTP 요청을 보내 요청한 웹페이지나 기타 리소스를 요청
               그런 다음 서버는 요청된 데이터가 포함된 HTTP 응답을 클라이언트에 다시 보낸다.
     HTTPS : SSL/TLS(Secure Sockets Layer/Transport Layer Security)암호화를 사용하여 클라이언트와 서버 간에 전송되는 데이터를 암호화하는 보다 안전한 HTTP 버전

   - TCP/IP를 사용하여 애플리케이션 구축
     포트 : 컴퓨터 네트워킹에서 물리적 또는 논리적 통신의 종착점(End Point)을 나타내는 개념으로 각 프로세스 또는 서비스에 고유하게 할당
              예를 들어, 웹 서버와 이메일 서버가 같은 IP 주소를 사용하더라도 서로 다른 포트 번호를 사용하여 데이터를 주고받습니다.
     소켓 : IP주소와 포트 번호의 조합으로 통신을 위한 특정 끝점을 나타냄 소켓은 장치 간 연결을 설정하고 애플리케이션 간에 데이터를 전송하는데 사용
     연결(Connection) : 두 장치가 서로 통신하려고 할 때 두 소켓 사이에 연결이 설정, 연결 설정 프로세스 중에 장치는 최대 세그먼트 크기, 창 크기와 같은 다양한 매개변수를 협상하여 연결을 통해 데이터 통신 방식을 결정
     데이터 전송 : 연결이 설정되면 각 장치에서 실행되는 애플리케이션 간에 데이터를 전송할 수 있다.

   - SSL/TLS로 인터넷 통신 보안
     SSL/TLS(Secure Sockets Layer/Transport Layer Security) : 인터넷을 통해 전송되는 데이터를 암호화하는 데 사용되는 프로토콜
     인증서 : 클라이언트와 서버 간의 신뢰를 구축할 때 사용 서버의 신원에 대한 정보가 포함되어 있으며 신뢰할 수 있는 제3자(인증기관)의 서명을 받는다.
     핸드셰이크 : 클라이언트와 서버는 보안 연결을 위한 암호화 알고리즘 및 기타 매개변수를 협상하기 위해 정보를 교환
     암호화 : 보안 연결이 설정되면 데이터가 암호화되며 클라이언트와 서버 간에 안전하게 전송할 수 있음
